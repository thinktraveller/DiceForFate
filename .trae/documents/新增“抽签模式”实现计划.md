## 目标
- 新增抽签模式（默认开启）：可选择 `1d2 / 1d3 / 1d4 / 1d6`，为每个点数填写对应事件；掷骰后显示“点数与事件”。
- 兼容现有普通掷骰与 `mdn` 功能；用户可随时切换模式。

## 交互与流程
- 顶部新增模式开关：`抽签模式`（Switch），默认开启；关闭时恢复现有普通掷骰逻辑。
- 抽签模式下：
  1) 选择骰子（`1d2 / 1d3 / 1d4 / 1d6`）。
  2) 弹出“事件配置”对话框，动态显示 N 个输入框（N=骰子面数），逐项填写事件文案；支持必填校验。
  3) 点击“开始抽签”→ 执行一次掷骰，显示“结果点数 + 对应事件”于主界面（居中、大字号）；将该次抽签记录写入历史。
- 历史界面：显示当次骰型、点数、事件文案与时间，仍支持清空。

## 数据模型
- 新增 `LotteryConfig(sides: Int, labels: List<String>)`。
- 历史项拓展显示：沿用 `RollResult`（记录点数与骰型），在渲染时按当次 `LotteryConfig` 找到对应事件；为确保历史可还原，新增：
  - 方案A：扩展模型 `RollResult` 增加可选字段 `event: String?`（推荐，避免双向耦合）。
  - 方案B：在 `HistoryStore` 追加同索引事件表（不推荐，易丢失）。

## 技术实现
- UI：
  - `MainActivity` 顶部新增 Switch；默认 `true`（SharedPreferences 持久化）。
  - 抽签事件配置对话框（`Material` Dialog + 动态纵向输入框列表，`TextInputLayout + TextInputEditText`）。
  - 主界面结果区：抽签结果用两行展示（第一行“点数”，第二行“事件”），当非抽签模式时维持原逻辑（m=1 单值 / m≥2 列表+总和）。
- 逻辑：
  - 当抽签模式开启且用户选择 `1dN`：
    - 若尚无有效配置，弹出配置对话框并校验所有输入非空；保存最近一次 `LotteryConfig`。
    - 生成点数后，查 `labels[rolled-1]` 写入 `event`，更新当前显示并写入历史。
  - 当抽签模式关闭：保持现有掷骰逻辑不变。
- 持久化：
  - `lottery_mode_enabled`（布尔，默认 true）。
  - 最近一次配置（可选）：`lottery_config_sides` 与 `lottery_labels_json`（便于复用与二次抽签）。

## 校验规则
- 事件配置：
  - 输入框数量 = 骰子面数；每项必填；可限制最大长度（如 64 字符）以防 UI 溢出。
  - “开始抽签”按钮仅在所有输入合法时启用。

## 代码改动点
- 新增：
  - `LotteryConfig.kt`（数据类）。
  - `LotteryConfigDialog`（可用函数式构建或内嵌于 `MainActivity`，负责渲染与校验）。
- 变更：
  - `MainActivity`：增加模式开关、事件配置与抽签流程、结果两行渲染（点数+事件）。
  - `HistoryActivity` / `ResultAdapter`：当存在 `event` 字段时在项中显示事件。
  - `HistoryStore` / `RollResult`：采用方案A为 `RollResult` 增加 `event: String?` 字段，并在写入历史时填充。

## 测试

  - 映射正确性：`sides=N` 时 `rolled=k` 映射到 `labels[k-1]`。
  - 历史保留：`event` 写入与展示正确。
- 手动验证：
  - 默认开启抽签模式；切换开关状态；配置与抽签流程；历史显示与清空。
